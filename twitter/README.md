# Twitter·降临派 | 自动化内容引擎

状态：架构设计中，待搭建

## 定位

**Twitter = 女娲第一个数字生命体的「嘴」**

与公众号的分工：
- 公众号（降临派手记）：深度长文，每周1-2篇，人主导
- Twitter（降临派）：快节奏短内容，每天1-3条，AI主导 + 人审核/微调
- 共享同一个知识库和选题池

---

## 整体架构

```
信息采集层（爬虫）
  ↓
Notion 知识库（结构化存储）
  ↓
内容引擎（Claude 选题 + 生成）
  ↓
发布层（Twitter API 自动发布）
  ↓
反馈层（读取 metrics → 写回 Notion）
  ↓
策略迭代（AI 根据数据调整选题和风格）
```

## 模块二：内容引擎

### Tweet 类型矩阵

| 类型 | 描述 | 频率 | AI自主度 |
|------|-----|------|---------|
| 🔥 热点速评 | 对当天AI新闻的快速观点 | 每天1条 | 高（AI选题+写+发） |
| 🧵 Thread深度 | 某个话题的5-10条Thread | 每周2条 | 中（AI写，人审核） |
| 💡 认知碎片 | 从女娲实践中提炼的洞察 | 每周2-3条 | 低（人提供素材，AI润色） |
| 🔄 转评 | 对大V推文的引用+评论 | 每天1条 | 高 |
| 📊 数据可视化 | AI行业数据图表 | 每周1条 | 中 |

### 内容生成 Prompt 框架

```
角色：你是「降临派」，一个相信AI将从根本上改变人类文明的技术深潜者。
风格：
- 不卖焦虑，讲结构
- 用类比让复杂事简单
- 有独立判断，不跟风
- 偶尔犀利，但有理有据
- 中英混用（适应AI Twitter圈的语言习惯）

输入：[知识库中的信息源]
输出：一条推文（<280字符）或一个Thread
要求：必须有一个「别人没说过的角度」
```

## 模块三：发布与反馈

### 技术栈

- **Claude Code**：核心调度引擎
- **Twitter API v2**：发布 + 读取 metrics
- **Notion API**：知识库读写
- **Cron Job（或云函数）**：定时触发

### 反馈指标

| 指标 | 用途 |
|------|-----|
| Impressions | 曝光量，衡量选题热度 |
| Engagement Rate | 互动率，衡量内容质量 |
| Follower Growth | 粉丝增长，衡量账号健康度 |
| Reply 内容 | 质性反馈，发现新选题 |

### 自主迭代逻辑

```
每周日 AI 自动执行：
1. 拉取过去7天所有推文的 metrics
2. 分析：哪类内容表现好/差
3. 生成「本周运营复盘」写入 Notion
4. 调整下周内容策略（类型比例、发布时间、话题偏好）
```

## 模块四：Notion 记忆层

### 需要创建的 Database

1. **信息源 Feed** — 爬虫采集的原始信息

3. **策略日志** — AI每周的策略调整记录
4. **写作风格库** — 表现好的推文样本，作为few-shot
