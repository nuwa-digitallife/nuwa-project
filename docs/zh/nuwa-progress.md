### 双循环架构（2026-02-28 确立）

**循环1：内容生产（当前焦点：跑通E2E）**

```
素材采集 → 选题推荐 → 深度调研 → 5-Pass引擎 → 配图 → 发布 → 反馈 → 回写经验 → 下一篇
```

状态：各模块已完成，待E2E端到端跑通。跑通后系统开始自转。

**循环2：能力生产（从循环1中涌现）**

循环1每次人手动干预的地方 = 循环2的种子。例：发现需要新监控源 → 系统自动生成能力提案 → 人确认 → 系统自己写代码部署。

落地方式：不提前设计。跑循环1时，每次手动干预记一条log。跑几周后log自己聚类，循环2结构涌现。

**核心原则**：循环1产出文章，循环2产出让循环1变强的代码和配置。先让循环1转起来，循环2从转的过程中长出来。

**旋涡模型**：系统一旦转起来，碎片化的时间、注意力、算力会被自动吸入。人的角色从"设计者"变为"投喂者"——从任何终端、任何时间、用最小成本把碎片扔进系统，系统自己分拣消化。

## 认知增强系统（write_engine/）

**状态**：✅ 代码完成，E2E 验证通过

6-Pass 引擎架构：
1. **Pass 1**：写作 Agent（叙事结构 + 标记待验证项）
2. **Pass 2**：事实验证 Agent（逐条核查 + 回退写作）
3. **Pass 3**：审视 Agent（隐性框架 + 完备性 + 春秋笔法检测）
4. **Pass 3.5**：协商闭环（6步：写作回应→事实回应→评估裁定→写作修改→事实修改→验收）
5. **Pass 5**：迭代求导（可选）— 证据硬度审计→定向调研→重写→版本对比→收敛判断
6. **Pass 4**：整合 Agent（合并修正 + 终稿输出）

与女娲框架的同构性：多 Agent 探索/压缩/回写 = 子项目 Agent 探索环境/压缩经验/回传母体。
