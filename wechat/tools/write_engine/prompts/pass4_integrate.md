## 任务

1. **合并检查**：确认协商共识中的所有 ✅ 修改都已体现在最新修订稿中
2. **叙事连贯性**：确保多轮修改没有破坏行文流畅度——修正一个错误可能导致上下文不通
3. **格式统一**：清理所有残留的 [待验证] 标记、重复内容、格式不一致
4. **生成全部交付物**

## 交付物清单

用分隔符输出以下全部内容：

```
===ARTICLE===
（终稿 article.md：完整文章，无 [待验证] 标记，无 [配图] 标记，格式干净）

===ARTICLE_MDNICE===
（mdnice 版 article_mdnice.md：和终稿内容完全一致，但图片 markdown 标记替换为 HTML 注释）
格式示例：
- 原文：![描述](images/xxx.png)
- mdnice版：<!-- 配图：描述 (images/xxx.png) -->
注意：正文不含 H1 标题（微信编辑器有独立标题栏，H1 会重复）

===POLL===
（投票 poll.md）
格式：
# 投票
问题：[顺着文章结尾情绪的一个问题]
1. [选项1]
2. [选项2]
3. [选项3]
4. [选项4]
注意：不设截止日期

===PUBLISH_GUIDE===
（发布指南 publish_guide.md）
内容：
# 发布指南：[文章标题]
## 文件清单
- [ ] article.md — 终稿
- [ ] article_mdnice.md — mdnice排版版
- [ ] review_report.md — 三层恰Review
- [ ] poll.md — 投票
- [ ] images/cover.png — 封面图（900x383，不含标题文字）
- [ ] images/ — 正文配图
## 简介（3个备选，每个≤70字）
1. ...
2. ...
3. ...
## 配图位置
（列出每张图对应文章中的哪个位置）
## 发布步骤
1. 将 article_mdnice.md 粘贴到 mdnice.com 排版
2. 从 mdnice 复制到微信编辑器
3. 逐个上传配图到对应位置
4. 设置封面图（900x383）
5. 填写简介（从上面3个备选中选）
6. 创建投票（从 poll.md 复制）
7. 勾选原创声明（文字原创）
8. 开启赞赏
9. 预览确认 → 保存草稿
10. 扫码发布
## 注意事项
- 封面图不放文字（微信会自动叠加标题）
- 投票不设截止日期
- 发布后无法补加投票，必须发布前确认

===DESCRIPTION_OPTIONS===
（3个简介备选，每个≤70字，不描述写作方法，给具体细节+核心判断+数字钩子）
1. ...
2. ...
3. ...
```
