## 核心事实（已验证）

### OpenClaw 是什么
- 开源 AI Agent 框架，由奥地利开发者 **Peter Steinberger** 创建（2025年11月）
- 本地运行的个人 AI 助手，通过 WhatsApp/Telegram/飞书/微信等聊天平台交互
- GitHub 250K+ stars（截至2026年3月4日），超越 React 成为 GitHub 历史最高星标项目之一
- 核心能力：读写文件、执行代码、浏览网页、收发邮件、管理日历、控制本地应用
- 信源：[GitHub 仓库](https://github.com/openclaw/openclaw), [Wikipedia](https://en.wikipedia.org/wiki/OpenClaw)

## 争议点和未解问题

### 1. 安全问题是根本性的还是可解决的？
- **结构性问题**：Skill 本质是在你的机器上运行的代码，ClawHub 发布门槛极低（仅需1周GitHub账号）
- **修补中**：OpenClaw 与 VirusTotal 合作扫描，但36%含prompt injection的数字触目惊心
- **哲学悖论**："让AI控制你的电脑"这个命题本身就存在信任问题

### 2. "全民养龙虾"是真需求还是泡沫？
- 很多人装完不知道让它干什么（触发文章开头自述）
- 大厂推广的实质是卖云算力——OpenClaw 是"Token粉碎机"，大量消耗 API token
- 付费装机 500元/次的"生意"可持续性存疑（闲鱼均价已降至37.38元）
- V2EX 帖子《全民 OpenClaw 狂热是不是一场噱头与泡沫？》直接质疑
- 36Kr：["第一批玩OpenClaw的人，已经开始清醒了"](https://36kr.com/p/3709855637598595)
- 有用户空闲 OpenClaw 每天消耗 $20 API 费用（心跳机制），每月 $750
- "删光邮件"和"自主购物付款"的失控事件被广泛引用
- Cisco："从能力角度看是突破性的；从安全角度看是绝对的噩梦"
- 信源：[V2EX](https://www.v2ex.com/t/1196421), [36Kr](https://36kr.com/p/3709855637598595)

### 3. 触发文章的商业利益
- **特工宇宙**已确认是商业 AI 媒体（粉丝12万+），业务包含与 AI 公司付费合作
- VoltAgent（awesome-openclaw-skills 维护方）是开发者工具公司，awesome-list 是常见的开源增长黑客手段——用精选资源引流
- 未发现特工宇宙与 VoltAgent 之间有直接商业关系的证据，但双方都有流量动机
- 文章推荐的项目本身真实且有价值，但**完全没有提及安全风险**是一个显著缺陷

### 4. Skill 标准之争
- Anthropic Agent Skills 规范 vs OpenClaw 的 SKILL.md 格式：本质相同（都是 Markdown+YAML frontmatter），但生态各自独立
- Vercel、OpenAI、Google ADK 各自也在推自己的 Agent 工具生态
- 最终会统一还是分裂？

### 5. Peter Steinberger 加入 OpenAI 的影响
- OpenClaw 移交基金会后，社区主导力量是否会变？
- OpenAI 是否会将 OpenClaw 商业化？Steinberger 承诺不会，但承诺归承诺

## Skill 技术架构（写作可用的解释素材）

### SKILL.md 结构
```yaml
---
name: my-skill
description: "What this skill does"
version: 1.0.0
metadata:
  openclaw:
    requires:
      env: ["API_KEY"]
      bins: ["node"]
---
# My Skill

## Instructions
（纯英文 Markdown 指令，Agent 按这些指令执行任务）

## Examples
（示例调用场景）
```

### 工作原理
1. Agent 启动 → 从文件系统加载 Skill 清单
2. 上下文组装阶段 → Skill 摘要（名称+描述+路径）注入 LLM prompt
3. LLM 判断当前任务匹配某 Skill → 按需读取该 Skill 的 SKILL.md 详细指令
4. **不是编译代码，是文本驱动架构**——Skill 是 Markdown 文档，Agent 读了照做

### 安装方式

## 与女娲项目的整合角度（用户指令相关）

用户指令："这个 skill 库怎么搞起来 纳入到女娲项目 这样我们所有的子项目能力都强了"

### 技术可行性分析
1. **Claude Code 已原生支持 SKILL.md**（`.claude/skills/` 目录），与 OpenClaw Skill 格式同源
2. 女娲项目当前已在使用 Skills 机制（如 `.claude/skills/analyze/SKILL.md`）
3. **直接可做**：
   - 从 awesome-openclaw-skills 筛选适合女娲子项目的 Skill，适配后放入 `.claude/skills/`
   - 为每个子项目（公众号/女娲核心/知识库等）建立专属 Skill 集
   - 将现有的自动化流程（如 publish、fetch-wechat）标准化为 SKILL.md 格式
4. **需要注意**：
   - OpenClaw Skill 面向 OpenClaw agent 写的指令，直接用于 Claude Code 需要适配
   - 安全审计很重要（36%含prompt injection）
   - 最好自研关键 Skill，第三方 Skill 仅做参考
