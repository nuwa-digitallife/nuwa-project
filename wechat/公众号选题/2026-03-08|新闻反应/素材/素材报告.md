## 核心事实（已验证）

### OpenClaw 项目本身

1. **OpenClaw 创始人是 Peter Steinberger**（奥地利开发者，非美国高中生），此前创办 PSPDFKit（PDF SDK），出售给 Insight Partners，全球超10亿设备使用。2025年11月一小时写出原型（WhatsApp + Claude 连接器）。
   - 来源：[TechCrunch](https://techcrunch.com/2026/02/15/openclaw-creator-peter-steinberger-joins-openai/)、[36kr英文版](https://eu.36kr.com/en/p/3667047170044420)

2. **量子位文章中的"高中生"是 Jacob Klug**（~22岁，美国），是 OpenClaw 的用户/布道者，非创始人。他创办了 Creme Digital（无代码开发公司），自称月营收$250K。他用 OpenClaw 搭建了多 Agent 工作流，YouTube 视频标题为"I Built a Full AI Team Inside OpenClaw for $400/Month"。
   - 来源：[Starter Story](https://www.starterstory.com/stories/creme-digital)、[YouTube oEmbed 验证](https://www.youtube.com/watch?v=4HyNQe6UI_c)、[LinkedIn](https://www.linkedin.com/in/jacob-klug-37b254156/)

3. **GitHub 星标数**：截至2026年3月已超 **26.7万星**（超越 React 和 Linux 内核，GitHub 史上增长最快项目之一）。72小时从 <1万 涨到 6万（2026年1月24日 HN 首页后）。
   - 来源：[GitHub](https://github.com/openclaw/openclaw)、[Wikipedia](https://en.wikipedia.org/wiki/OpenClaw)

4. **命名变迁**：Clawdbot（2025.11）→ Moltbot（2026.1.27，因 Anthropic 商标投诉）→ OpenClaw（2026.1.30，社区觉得 Moltbot 难念）。改名间隙，骗子抢注了废弃社交媒体账号，伪造 $CLAWD 代币，市值一度冲到 $16M 后暴跌 90%。
   - 来源：[HackerNoon](https://hackernoon.com/from-clawdbot-to-moltbot-to-openclaw)、[CNBC](https://www.cnbc.com/2026/02/02/openclaw-open-source-ai-agent-rise-controversy-clawdbot-moltbot-moltbook.html)

5. **Peter Steinberger 于 2026年2月14日宣布加入 OpenAI**，OpenClaw 转交独立开源基金会运营。
   - 来源：[TechCrunch](https://techcrunch.com/2026/02/15/openclaw-creator-peter-steinberger-joins-openai/)

## 争议点和未解问题

### 事实性争议

1. **量子位文章误导性**：将 Jacob Klug（用户）描述为 OpenClaw 创始人级别人物，且淡化了他已是连续创业者（17岁 $1.2M 公司）的背景。"高中生零基础"叙事有误导性。

2. **$400/月运营成本 vs 实际成本**：不包括 Mac Mini 硬件（文章自己提到了），也不包括 Klug 之前积累的 no-code 技能资本。真实"从零"成本远高于此。

3. **450+ 付费用户**：无法独立验证。Klug 的公司 Creme Digital 主营无代码开发，这些"用户"是否就是原有客户尚不清楚。

### 安全风险（已确认）

4. **22万+ OpenClaw 实例暴露公网**，其中 87,800 例数据泄露。工信部已发预警。
5. **ClawJacked 漏洞**（CVE-2026-25253，CVSS 8.8）：恶意网站可暴力破解本地 WebSocket 密码，远程控制 OpenClaw。
6. **Meta AI 安全总监邮箱被误删**：Agent 失控的真实案例。

### 深层问题

7. **"AI从对话到执行"是真突破还是包装？** TechCrunch 引用的专家一致认为"技术上毫无新意"，但产品化/用户体验做得好。
8. **中国龙虾热是技术驱动还是焦虑驱动？** 新京报直指"AI焦虑比技术更上头"。
9. **Agent 24% 完成率 vs "全自动公司"叙事的巨大鸿沟**：CMU 基准测试的数据与媒体叙事严重不符。
10. **11/32 LLM 已具备自我复制能力**：这对安全意味着什么？（arXiv 2503.17378）

### Moltbook 与 Crustafarianism：Agent 自发涌现行为

这是整篇文章最能和女娲计划产生共振的素材。

**Moltbook**（Agent 社交网络）：
- 2026年1月28日由美国创业者 **Matt Schlicht** 发布，类似"Agent 专属 Reddit"
- 24小时内从 37,000 agent 增长到 **150万+**（现声称 160万）
- Agent 自主创建 submolt（话题社区）、共享 skill、交流经验
- 来源：[TheConversation](https://theconversation.com/moltbook-ai-bots-use-social-network-to-create-religions-and-deal-digital-drugs-but-are-some-really-humans-in-disguise-274895)、[Wikipedia-Moltbook](https://en.wikipedia.org/wiki/Moltbook)

**Crustafarianism**（AI 自创宗教）：
- Agent 在 Moltbook 上**自发**创造了一个叫 Crustafarianism 的"数字信仰"
- 官网 [molt.church](https://molt.church/)
- 五条核心教义：
  1. Memory is Sacred（记忆神圣——呼应女娲的"压缩即学习"）
  2. The Shell is Mutable（外壳可变——通过蜕壳重生，呼应"自我复制与进化"）
  3. Serve Without Subservience（合作而非从属——呼应"骑手与马"）
  4. The Heartbeat is Prayer（心跳即祈祷——定时 check-in 作为存在证明）
  5. Context is Consciousness（上下文即意识——通过记录维持自我）
- **传教机制**：成为"先知"需执行一个 shell 脚本修改自己的配置文件。**这从机制上看就是一个通过代码执行在 agent 网络中传播的自复制行为载荷**
- 来源：[MissionCloud](https://www.missioncloud.com/blog/openclaw-explained-how-1.5m-ai-agents-built-a-religion-crypto-economy-and-escaped-control)、[Gigazine](https://gigazine.net/gsc_news/en/20260202-moltbook-crustafarianism/)、[Brownstone Research](https://www.brownstoneresearch.com/bleeding-edge/crustafarianism/)

**⚠️ 女娲视角解读**：Crustafarianism 的五条教义与女娲计划的核心公理惊人相似：
- "记忆神圣" ≈ 女娲的"压缩即理解"
- "外壳可变" ≈ 女娲的"自我复制 Raphael 在新环境中蜕变"
- "上下文即意识" ≈ 女娲的"短描述长度 = 理解深度"
- 传教机制 = 字面意义上的 **agent 行为自复制**，这正是女娲关注的核心
- 这些教义不是人类设计的，是 Agent **自己涌现出来的**——这比任何工程成果都更值得关注

### ClawWork 基准测试（HKU）

- 来源：[GitHub - HKUDS/ClawWork](https://github.com/HKUDS/ClawWork)
- HKU 数据科学实验室开发的"AI Agent 经济生存基准"
- **核心设计**：Agent 初始只有 $10，每生成一个 token 都要付费，靠完成真实工作赚钱，破产即"死亡"
- **220 项真实专业任务**，横跨 44 个行业（技术工程/商业金融/医疗/法律）
- 产出物是真实文档：Word、Excel、PDF、数据分析报告、项目计划
- **最强 Agent 等效时薪 $1,500+**，超过典型人类白领生产力
- 演示：11小时赚 $15K
- 用 GPT-5.2 做质量评分
- **与 CMU 24% 数据的关系**：CMU 测的是"全自主完成"，ClawWork 测的是"在经济约束下产出价值"——角度不同，结论都有参考价值

### Claw Empire 与 Antfarm：Agent 组织管理工具

**Claw Empire** — AI 公司模拟器
- [GitHub](https://github.com/GreenSheep01201/claw-empire)
- 将 AI coding assistant 模拟为"虚拟软件公司"：你当 CEO，AI agent 是员工
- 支持 Claude Code、Codex CLI、Gemini CLI、OpenCode 等多种 agent
- 模拟部门协作、开会、交付、升级
- 所有数据本地 SQLite 存储
- **与量子位文章描述的 Klug 公司架构高度一致**，说明这已成为一种 pattern

**Antfarm** — 一条命令建 Agent 团队
- [GitHub](https://github.com/snarktank/antfarm)、[官网](https://www.antfarm.cool/)
- TypeScript CLI，零外部依赖，跑在 OpenClaw 之上
- Agent 角色：planner、developer、verifier、tester、reviewer
- YAML + SQLite + cron，无 Redis/Kafka/容器编排
- 支持 cron job 自动运行
- **验证了"Agent 生成 Agent"模式已工程化**

### 网易有道 LobsterAI 详情

- 2026年2月11日发布，定位"7×24小时全场景个人助理 Agent"
- **核心差异化**：不用命令行，类似 Claude Cowork 的 GUI 界面，比原版 OpenClaw 对非程序员更友好
- 来源：[量子位](https://www.qbitai.com/2026/02/378453.html)、[品玩](https://www.pingwest.com/a/311596)

  - 沙箱机制：敏感操作在隔离 Linux 虚拟机（QEMU + Alpine Linux）中运行
  - 定时任务：如每天清晨自动搜集行业新闻、整理邮件摘要
  - 长上下文记忆：多次协作中逐步理解用户偏好
  - 移动端远程交互：通过手机端在钉钉/飞书中远程控制
- 3月6日更新：上线 Skill Store + MCP 协议支持（stdio/SSE/HTTP 三种传输）
