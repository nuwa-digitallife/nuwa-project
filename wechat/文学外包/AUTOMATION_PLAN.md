# 文学外包流水线 · 项目计划

> **这是文学外包项目的 single source of truth。** 每次会话开始先读这个文件，做完一步就更新状态。
> 丢了 context 不要紧，读这个文件就能接上。
>
> 快速参考卡片（笔名/刊物/投稿矩阵）→ [`档案.md`](档案.md)

## 与公众号项目的关系

公众号 = 科普赛道（循环1：选题→写作→发布→反馈）。文学外包 = 文学赛道（征稿→金线写作→投稿→采用反馈）。两条线并行，共享底层能力（engine.py 核心函数、素材采集工具）。

文学线独特价值：
- 外部验证金线方法论——被专业刊物采用 = 方法论 work 的硬证据
- 多体裁实验——散文→小说→诗歌，验证金线法适用边界
- 积累发表记录

## 架构总览

```
[素材库积累]              ← 素材库/{域名}/README.md（手动 + 写作后回填）
       ↓
[征稿机会匹配]            ← 投稿机会/机会数据库.md（月度更新）
       ↓
[人工：选题确认]           ← 人决策：这个征稿值得写吗？
       ↓
[Pass 0 金线压缩]         ← lit_write.py --pass 0（交互式确认）
       ↓
[人工：金线确认]           ← 人决策：这个意象对了吗？（刻意保留）
       ↓
[Pass 1 展开写作]         ← lit_write.py --pass 1
[Pass 2 金线自检+败相扫描] ← lit_write.py --pass 2
[Pass 3 整合修订→终稿]    ← lit_write.py --pass 3 → final.md + DOCX
       ↓
[邮件草稿生成]            ← tools/draft_email.py（Gmail IMAP → 草稿箱）
       ↓
[人工：检查草稿→发送]     ← 人决策（保留，不做全自动发送）
       ↓
[投稿跟踪]               ← 已投稿/submissions.jsonl
       ↓
[状态汇总]               ← lit_status.py
       ↓
[反馈回传]               ← 采用/拒稿 → 更新方法论 + 素材库 + 策略
       ↓
 ↺ 回到 [征稿机会匹配]
```
