### **Agent 开始变得「主动」**

回头看，AI 领域变化的速度快得惊人。

一年前，开发者社区讨论最多的还是 Cursor 和 Windsurf。它们的核心体验是人发布任务，AI 辅助写代码。本质上，人类仍然是决策者，AI 只是一个刚入门的实习生。

但到了 Claude Code 和 OpenClaw，性质发生了质的转变。用户开始完全信任 Agent 来替自己做决策。你可以同时启动四五个 Agent，在它们之间切换，但已经不再逐行审查它们的输出了。

**Agent 自己决定要做什么、怎么做、用什么工具来做。**

YC 的 CEO Garry 分享了一个亲身经历：他用 Claude Code 在两周内复现了此前一个创业团队花了好几年才完成的产品原型。

![](https://mmbiz.qpic.cn/mmbiz_png/nR18mzFEw1EGibVrmdEkibCLOzA0SoOZsC4E7dbDicHwUIwYFjrIiaPJdUJFOs6ictT4Ehmz2K4MVwXVic3FnWoks7D9iaIwupWBRjecxfwcvfUed4/640?wx_fmt=png&from=appmsg)

当 Garry 需要对视频内容做文字转录时，Claude Code 自动为他选择了 Whisper 模型。但，它选的是好几年前的 Whisper V1 版本，因为那个版本的 API 文档在网上最容易被找到。结果，处理一小时的视频真的需要整整一小时，完全没有速度优势。

后来 Garry 发现，换成 Grok 可以快 200 倍，成本还低 10 倍。

从另一个角度看，这也说明当下的技术水平仍然存在大量改进空间。

正如 Garry 自己总结的：**如果 Agent 已经完美，就不会犯这种错误了。**

### **面向 Agent 的 GEO**

在传统的开发者工具市场里，产品获客的方式我们都很熟悉：

开发者在社区里互相推荐，在 GitHub 上看 star 数，在 Stack Overflow 上搜解决方案，工具的口碑靠人际传播来建立。

但当 Agent 成为工具的选择者和使用者时，这套逻辑被 GEO 彻底改写了。

产品的使用文档，成了 Agent 选择开发工具最重要的「销售」。

YC 合伙人 Harj Taggar 举了一个例子：过去 12 个月里，Postgres 数据库的创建量出现了爆发式增长。很大程度上是因为各种 AI Agent 在帮用户搭建应用时，默认选择了 Supabase。
